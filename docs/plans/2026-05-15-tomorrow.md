## 2. Rezervačný formulár — slot picker UX prerábka

**Problém ktorý si identifikoval:**
- Chips ukazujú start times v 30-min krokoch („09:00", „09:30", „10:00", …)
- Lenže každý balíček má rôzne trvanie (MINI 2h, MAXI 3h, CLOSED 4h)
- Z UI to vyzerá, že sloty sú 30-minútové → mätúce dokonca pre developera
- Pričom z homepage je jasné, že oslavy sú viachodinové balíčky

**Aktuálne správanie (technicky správne, vizuálne nezrozumiteľné):**
- Algoritmus vyberie všetky 30-min start times kde sa zmestí celý balíček
- Klient klikne na 09:00 pre MINI → backend vytvorí rezerváciu 09:00–11:00
- Žiadne vizuálne info o tom, že to zablokuje aj 09:30 a 10:00 v slot pickeri

**Návrh UX (na zvyklotnenie zajtra):**

Možnosť **A — chip ukazuje full range:**
```
[09:00 – 11:00 (2h)] [09:30 – 11:30 (2h)] [10:00 – 12:00 (2h)] …
```
Plus: text nad slot pickerom „Trvanie balíčka: **2 hodiny**".

Možnosť **B — visual "block" effect po klik:**
- Chips ostávajú zobrazené ako start times „09:00", „09:30", „10:00", …
- Po kliknutí napr. 09:00:
  - 09:00 sa vyfarbí ako selected (accent)
  - 09:30, 10:00, 10:30 sa **vizuálne striked-out / faded** so štítkom „súčasť vašej rezervácie"
  - Pod gridom „Vybraný termín: **09:00 – 11:00** (2 hodiny, MINI)" badge
- Plus help text nad: „Vyberte začiatočný čas. Pokryje 2 hodiny."

Možnosť **C — timeline view namiesto chips:**
- Vodorovná os 09:00 – 20:00 s farebne odlíšenými blokmi
- Voľné, plné, vybraný — všetko viditeľné naraz
- Časové bloky majú dĺžku úmernú duration

**Moje odporúčanie:** kombinácia **A + B** — chip má dlhý label „09:00–11:00" a po výbere sa zvyšné chips visually zoslabia s tooltipom „v rovnakom časovom okne ako vaša rezervácia". Plus badge „Vybraný termín: 09:00–11:00 (2h MINI)" v step__lead časti.

**Pred kódením treba rozhodnúť:**
- Ktorá z A/B/C/A+B
- Či má časový rozsah byť aj na **kalendárových bunkách** (dnes ukazujú „19×" = počet slotov, mohlo by ukazovať „MINI 9–18" alebo nič)
- Či dať explicit „Trvanie balíčka" badge pred slot grid

**Code touch (ak A+B):**
- `private/templates/pages/reservation.php` — Step 2 → pridať „Trvanie balíčka" badge a „Vybraný termín" badge
- `public/assets/js/rezervacia.js` — `renderSlots(slots)` upraviť aby každý chip renderoval „START–END (Xh)", po `selectDay` zachytiť `duration` z atribútu balíčka, po `slot.addEventListener('click')` doplniť visual block-effect na ostatné chips (kalkulovať či sa prekrývajú so selected range)
- `public/assets/css/rezervacia.css` — pridať `.slot.is-blocked-by-selection` štýl (faded + striked-out)

**Estimate:** 1–2 h pre A+B, 3+ h pre C (timeline).

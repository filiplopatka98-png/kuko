# Admin IA v2 + content/gallery changes — plan

> Execute via subagent-driven-development (implementer → spec review → code-quality review per task), deploy at end. WP-style admin. Builds on the existing sidebar layout (`private/templates/admin/layout.php`) + routes (`public/admin/index.php`). Routes/handlers stay unless noted; mostly REGROUPING + 2 new editors + 1 gallery feature + global input styling.

## User decisions (final)
- **Stránky** = list of public pages, each editable: **content + SEO together**. Pages: Domov, Rezervácia, Fotogaléria, Časté otázky, Ochrana údajov. FAQ + Ochrana údajov get **editable body** (migrate hardcoded template text → content blocks).
- **Balíčky** → tab under **Rezervácie**.
- **Kontakt** (site-wide contact+social) → tab under **Nastavenia**.
- **Galéria** → own sidebar item (media manager) + NEW: choose which photos show on homepage via checkboxes, **max 6**; if fewer selected, fill remaining randomly from other visible photos.
- **A1**: reservation settings (`/admin/settings`: buffer/horizon/lead/slot) → tab under **Rezervácie** (label "Nastavenia rezervácií").
- **A3**: Maintenance + Logy + GDPR → tabs under **Nastavenia**.
- **A4**: every `<input>/<select>/<textarea>` in the whole admin styled consistently (like the contact `.admin-field` inputs).

`$isResvGroup` = path in {/admin,/admin/reservation*,/admin/calendar,/admin/blocked-periods,/admin/opening-hours,/admin/packages,/admin/settings}. New `$isSettingsGroup` = {/admin/contact,/admin/maintenance,/admin/log,/admin/gdpr}. New `$isPagesGroup` = {/admin/pages*}. Reuse normalized `$path` (rtrim) + `$active`/`$aria` helpers.

## Page→content/SEO mapping (for Stránky editor)
- **Domov** (`/`, pageType `home`): content groups hero/about/cennik/kontakt/footer (existing content_blocks) + `seo.home.*`. (`oslavy`/packages NOT here — packages live under Rezervácie.)
- **Rezervácia** (`/rezervacia`, `rezervacia`): `seo.rezervacia.*` (+ optional intro block `rezervacia.intro` — only if trivially present; otherwise SEO-only).
- **Fotogaléria** (`/galeria`, `gallery`): `seo.gallery.*` (+ existing galéria heading/lead are content blocks? if not, SEO-only; photos managed in Galéria).
- **Časté otázky** (`/faq`, `faq`): NEW editable FAQ content blocks + `seo.faq.*`.
- **Ochrana údajov** (`/ochrana-udajov`, `privacy`): NEW editable privacy body block(s) + `seo.privacy.*`.
